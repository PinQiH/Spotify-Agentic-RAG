# 🎯 Spotify Agentic RAG: 業務核心邏輯說明

本文件旨在闡述 **Spotify Agentic RAG** 系統背後的業務邏輯與設計哲學。不同於傳統專注於數值計算的推薦系統，本系統的核心價值在於提供具備「理解能力」與「溝通能力」的音樂推薦服務。

---

## 🚀 1. 核心願景：從「算命」到「專家對話」

### 傳統推薦的痛點

傳統推薦系統（如基於矩陣分解的協同過濾）通常像一個「黑箱」。它告訴你「因為你聽過 A，所以你可能喜歡 B」，但卻無法解釋具體原因。這導致：

- **信任感缺失**: 使用者不明白為什麼被推薦這首歌。
- **冷啟動困難**: 缺乏歷史數據時，系統無法理解使用者的當前意圖。

### Agentic RAG 的解決方案

我們將 LLM 引入推薦核心，使其扮演一位 **「精通音樂心理學的數位 DJ」**：

- **可解釋性**: 每首推薦歌曲都附帶具體的「推理理由」。
- **場景感知**: 透過 Agent 理解使用者的「人格畫像」與「當前氛圍」。

---

## 👥 2. 使用者畫像 (Persona) 體系

系統不將使用者視為隨機的 ID，而是將其抽象化為特定的 **音樂人格 (Persona)**。這套體系是業務邏輯的基石：

| 畫像名稱               | 音樂品味特質                 | 業務目標場景             |
| :--------------------- | :--------------------------- | :----------------------- |
| **Chill Vibes**        | 低能量、高聲學感、放鬆。     | 居家休閒、午後閱讀。     |
| **Party Animal**       | 高能量、強烈舞動感、快節奏。 | 派對、社群聚會。         |
| **Study Focus**        | 純樂器、穩定、中低節奏。     | 深度工作、學習專注。     |
| **Workout Motivation** | 極高能量、強勁重低音。       | 健身、跑步、高強度運動。 |

### 長期偏好總結 (Persona Summary)

系統會對畫像的歷史收聽紀錄進行「語義化總結」，形成一份數位檔案（如：「該使用者偏好清脆的吉他聲與具有空靈感的女性人聲」）。這讓 Agent 在做決定時，有一個明確的「品味標準」。

---

## 🌉 3. 語義橋接：連結音訊數據與人類感性

音樂的本質是數值訊號（Energy, Tempo），但人類的聽感是感性的語義（「這首歌聽起來很熱血」）。

### MLP Soft-Prompting 的業務角色

我們訓練了一個 **多層感知器 (MLP)**，其任務是將 11 項音訊特徵映射到 384 維的語義空間：

- **輸入**: [Energy: 0.8, Tempo: 128, ...] (機器語言)
- **輸出**: [Semantic Embedding] (對應到「充滿活力、適合跳舞」的文字意義)

這座橋樑讓「硬邦邦的數據」能夠參與到「感性的 LLM 推理」中。

---

## 🧠 4. Agent 推理決策鏈 (Logic Flow)

當系統進行推薦時，它遵循以下業務邏輯鏈條：

1. **感知 (Perception)**: 讀取當前播放歌曲的語義特徵。
2. **檢索 (Retrieval)**:
   - **快速召回**: 從預計算清單中找相似歌曲。
   - **語義召回**: 使用預測出的 Embedding 從 FAISS 找「聽起來感覺對」的歌曲。
3. **過濾 (Filtering)**: 移除使用者聽過的歌曲或顯式內容。
4. **推理 (Reasoning)**: 將候選集交給 LLM，要求其：
   - 「請根據此使用者的 **Workout** 畫像，從候選中挑出最能激發能量的歌。」
   - 「並引用歌曲描述中的 **『重低音』** 特質說明理由。」

---

## 📈 5. 持續優化之反饋迴圈 (Feedback Loop)

系統並非靜態。透過整合 **Google Sheets API**，我們建立了一個閉環的反饋系統：

1. **收集 (Collect)**: 紀錄使用者對推薦結果的「喜歡/不喜歡」投票。
2. **分析 (Analyze)**: 比對使用者偏好與 Agent 推理理由的一致性。
3. **進化 (Evolve)**:
   - **Prompt 工程**: 修正 Agent 的推薦策略。
   - **模型微調**: 利用點擊數據重新訓練 MLP，讓它更理解特定的品味趨向。
